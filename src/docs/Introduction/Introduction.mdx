# CoConstruct Design System

This is the central library for the user interfaces (UI) of all digital products built and maintained by CoConstruct. It provides reusable components, patterns, foundations, and guidelines for teams designing and developing experiences across web applications and marketing sites.

The CoConstruct Design System enables teams to move faster, collaborate more effectively, and deliver consistent, high-quality experiences for builders, remodelers, and their clients. By aligning design and engineering around shared standards, we ensure that everything we create reflects our brand, supports usability, and builds trust with construction professionals who rely on our platform every day.

By using the system across all digital touchpoints, our products:

- Look consistent and cohesive
- Behave predictably and intuitively
- Meet accessibility standards
- Scale efficiently as the platform evolves.

## How It Works

The most tangible outputs of the CoConstruct Design System are reusable UI components—buttons, inputs, dropdowns, navigation elements, cards, tables, modals, and more.

These components:

- Follow CoConstruct’s brand foundations (color, typography, spacing, iconography)
- Adapt responsively across screen sizes and devices
- Are built with accessibility in mind
- Support real construction workflows and data-heavy interfaces

Each component is documented with usage guidance, states, accessibility notes, and implementation details to ensure consistency from design to production.

---

### Design Foundations

Our system is built on a clear set of foundations:

- **Design Tokens** – Colors, spacing, typography scales, elevation, motion, and border radius values defined as constants and semantic tokens
- **Accessibility Standards** – Contrast ratios, keyboard support, focus states, and screen reader considerations
- **Interaction Principles** – Clear hierarchy, feedback, and predictable behavior
- **Layout System** – Grid, responsive breakpoints, and spacing rules

These foundations ensure flexibility while maintaining consistency across teams.

---

### Figma Library

Our Figma library provides:

- Production-ready components
- Text styles and typography scales
- Color styles and semantic tokens
- Grid and layout systems

Designers can compose screens quickly using shared components that directly map to coded implementations. Components are versioned and maintained to reflect the current production state.

---

### React Library

On this site, you’ll find the React component library and documentation.

Developers can:

- Install components via npm
- Reference live examples and usage guidelines
- Follow accessibility and implementation best practices
- Leverage design tokens directly in code

Each component is built to be composable, scalable, and aligned with real product workflows in the CoConstruct platform.

---

### CSS & Design Tokens

The design system is framework-flexible.

Teams may use:

- CSS classes
- Design tokens
- Utility-based approaches

This ensures the system can support multiple product surfaces and technical stacks while preserving visual and behavioral consistency.

## Why It Matters

CoConstruct serves builders managing complex projects, client communication, budgeting, and scheduling. Clarity, trust, and efficiency are essential.

The design system helps us:

- Reduce design and engineering redundancy
- Improve onboarding for new team members
- Deliver features faster
- Maintain brand integrity
- Create predictable, confidence-inspiring experiences

Consistency in UI leads to confidence in the product.
